# LSEVGG v3 模型架构详解

---

## 一、宏观全貌

```

### 数据流一句话总结

> 一张 224×224 的 RGB 遥感图像进入网络，经过 Stem 提取 64 通道浅层特征 → 5 个 Stage 逐步压缩空间尺寸、扩张通道数 → GAP 把 7×7 的特征图"压扁"成一个 512 维向量 → 分类头将这个向量映射为 45（或 15）个类别的分数。

## 二、逐模块拆解

### 2.1 Stem：浅层特征提取

```
输入: [B, 3, 224, 224]
          │
          ▼
    Conv2d(3 → 64, kernel=3, stride=1, padding=1)
          │   作用: 将 RGB 3 通道扩展为 64 个特征通道
          │   计算: 每个输出像素 = 3×3×3=27 个参数的加权和
          │   参数量: 3×64×3×3 = 1,728 个
          ▼
    BatchNorm2d(64)
          │   作用: 对每个通道做标准化，(x-μ)/σ，稳定训练
          │   参数量: 64×2 = 128 个 (γ 和 β)
          ▼
    SiLU 激活
          │   作用: y = x · σ(x)，非单调平滑激活
          │   好处: 负值不直接清零，保留微弱梯度
          ▼
输出: [B, 64, 224, 224]
```

**为什么 Stem 使用标准卷积而非深度可分离卷积？**

Stem 是网络的"第一印象"，此时输入只有 3 个通道，信息高度密集。标准卷积可以在 3×3 区域内同时混合 RGB 三通道的颜色信息与空间纹理信息，建立丰富的浅层特征。如果在此处使用深度可分离卷积（group=3），每个输出通道只能看到一种颜色，会丢失关键的跨通道颜色关系。

### 2.2 Stage 1~5：特征提取主干

每个 Stage 的结构完全相同：

```
Stage N = ResidualBlock(s) + MaxPool2d(2)
```

**MaxPool2d(2) 的作用：**

```
输入: [B, C, H, W]
       │
       ▼  在每个 2×2 窗口内取最大值，步长 2
输出: [B, C, H/2, W/2]

效果:
  · 空间尺寸减半  (224→112→56→28→14→7)
  · 每个像素的感受野扩大一倍
  · 计算量减半（下一层处理的数据更少）
  · 引入平移不变性
```

**5 个 Stage 的参数总览：**

```
┌──────────┬───────────────┬─────────────┬──────────────┬──────────────┐
│  Stage   │   包含的块     │  输入尺寸    │   输出尺寸    │  通道变化     │
├──────────┼───────────────┼─────────────┼──────────────┼──────────────┤
│ Stage 1  │ 1×ResBlock    │ 224×224×64  │ 112×112×64   │  64 →  64   │
│          │ + MaxPool     │             │              │             │
├──────────┼───────────────┼─────────────┼──────────────┼──────────────┤
│ Stage 2  │ 1×ResBlock    │ 112×112×64  │  56×56×128   │  64 → 128   │
│          │ + MaxPool     │             │              │             │
├──────────┼───────────────┼─────────────┼──────────────┼──────────────┤
│ Stage 3  │ 2×ResBlock    │  56×56×128  │  28×28×256   │ 128 → 256   │
│          │ + MaxPool     │             │              │             │
├──────────┼───────────────┼─────────────┼──────────────┼──────────────┤
│ Stage 4  │ 2×ResBlock    │  28×28×256  │  14×14×512   │ 256 → 512   │
│          │ + MaxPool     │             │              │             │
├──────────┼───────────────┼─────────────┼──────────────┼──────────────┤
│ Stage 5  │ 2×ResBlock    │  14×14×512  │   7×7×512    │ 512 → 512   │
│          │ + MaxPool     │             │              │             │
└──────────┴───────────────┴─────────────┴──────────────┴──────────────┘
```

**设计哲学：空间↓，通道↑**

这是一个经典的 CNN 设计模式：当空间分辨率减半时，通道数翻倍，使得总信息容量（C × H × W）大致保持稳定：

```
Stage 0 (Stem): 64 × 224 × 224 = 3,211,264 个值
Stage 1:        64 × 112 × 112 =   802,816
Stage 2:       128 ×  56 ×  56 =   401,408
Stage 3:       256 ×  28 ×  28 =   200,704
Stage 4:       512 ×  14 ×  14 =   100,352
Stage 5:       512 ×   7 ×   7 =    25,088
```

信息从"宽而浅"（大面积、少通道）逐步转变为"窄而深"（小面积、多通道）。浅层捕获空间细节（边缘、纹理、角点），深层捕获语义信息（物体类别、场景结构）。

### 2.3 ResidualBlock：LSEVGG 的核心构件

这是整个模型最重要的模块，每个 Stage 都由 1~2 个 ResidualBlock 组成。

```
ResidualBlock(in_ch, out_ch)

                    ┌─────────── 主路径 (Main Path) ───────────┐
                    │                                           │
   x ───────────────┤                                           ├──→ 输出
   [B, in_ch, H, W] │                                           │
       │            │                                           │
       │            │  DSC1              DSC2              SE   │
       │            │  ┌─────┐         ┌─────┐         ┌─────┐  │
       ├────────────┼─→│ DSC │────────→│ DSC │────────→│ SE  │──┤
       │            │  │in→out│        │out→out│       │     │  │
       │            │  └─────┘         └─────┘         └─────┘  │
       │            │                                           │
       │            │                                           │
       └─── 跳跃连接 (Skip Connection) ───┘                     │
            如果 in_ch ≠ out_ch: Conv1×1(in_ch→out_ch)           │
            如果 in_ch = out_ch: 直通 (Identity)                 │
                                                                 │
                                      输出 = SiLU(主路径 + 跳跃连接)
                                      [B, out_ch, H, W]
```

#### 一步一步走通 ResidualBlock.forward()：

```
第 1 步: 保存跳跃连接
  identity = self.skip(x)
  
  如果 in_ch == out_ch:  identity = x  (什么都不做，原样保留)
  如果 in_ch != out_ch:  identity = Conv1×1(x) → BN  (对齐通道数)
  
  例如 Stage 2 中 in_ch=64, out_ch=128:
    1×1 卷积创建 128 个通道，每个通道是 64 个输入通道的线性组合
    1×1 卷积参数量 = 64×128×1×1 = 8,192 个

第 2 步: 第一个深度可分离卷积
  out = self.dsc1(x)   # in_ch → out_ch
  
  做两件事:
    (a) 对每个输入通道独立做 3×3 卷积（空间混合）
    (b) 用 1×1 卷积将通道混合到目标数量（通道混合）
  输出: [B, out_ch, H, W]

第 3 步: 第二个深度可分离卷积
  out = self.dsc2(out)  # out_ch → out_ch
  
  在第一个 DSC 的基础上做进一步的特征精炼
  注意: 这次 in_ch=out_ch，不会改变通道数
  输出: [B, out_ch, H, W]

第 4 步: SE 通道注意力
  out = self.se(out)
  
  给 128/256/512 个通道分别打分（0~1 之间的权重）
  重要的通道权重接近 1，噪声通道权重接近 0
  输出: [B, out_ch, H, W]  (尺寸不变，但通道被重新加权)

第 5 步: 残差相加
  out = out + identity
  
  将主路径学到的"残差"（即需要改进的部分）
  与跳跃连接传过来的"原始信息"相加
  
  直觉理解:
    主路径  = 学习"输入还需要补充什么特征"
    identity = 把"已有的特征"直接传过来
    out + identity = 在已有特征的基础上叠加新学到的特征

第 6 步: 最终激活
  out = self.act(out)  # SiLU
  
  对融合后的特征做非线性变换
  输出: [B, out_ch, H, W]
```

**为什么要用残差连接（跳跃连接）？**

```
没有残差的网络:
  梯度: loss → LayerN → ... → Layer2 → Layer1
  问题: 每经过一层，梯度乘一个小于 1 的数
       到浅层时梯度 ≈ 0 → 浅层权重无法更新 → 学不到细粒度特征

有残差的网络:
  梯度: loss → LayerN → ... → Layer2 → Layer1
              └────── shortcut ──────┘ (梯度直通)
  效果: 即使中间层梯度衰减，跳跃连接确保梯度无损到达浅层
       浅层也能有效学习 → 可以区分 small-vehicle 和 large-vehicle 的细微差异
```

### 2.4 DepthwiseSeparableConv（DSC）：参数压缩的引擎

标准卷积 vs 深度可分离卷积的对比：

```
┌──────────────────────────────────────────────────────────────────┐
│                    标准卷积 (Standard Conv2d)                     │
│                                                                  │
│  输入: [B, C_in, H, W]    卷积核: C_out 个 [C_in, 3, 3]         │
│                                                                  │
│  每个输出通道 = 所有输入通道的 3×3 加权求和                        │
│                                                                  │
│  ┌───┐                                                            │
│  │▓▓▓│  3×3 卷积核 (C_in 层)                                      │
│  │▓▓▓│  第 1 个输出通道 = Σ(输入通道_i ⊗ 卷积核_i)                │
│  │▓▓▓│  第 2 个输出通道 = Σ(输入通道_i ⊗ 卷积核_i)                │
│  └───┘  ... 共 C_out 个                                           │
│                                                                  │
│  参数量: C_in × C_out × 3 × 3                                    │
│  例如 64→128: 64×128×9 = 73,728                                  │
├──────────────────────────────────────────────────────────────────┤
│               深度可分离卷积 (DepthwiseSeparableConv)              │
│                                                                  │
│  第 1 步: Depthwise (逐通道)                                     │
│  ┌───┐                                                            │
│  │▓▓▓│  每个通道用自己的 3×3 卷积核                               │
│  │▓▓▓│  输入通道 0 → 中间通道 0                                   │
│  │▓▓▓│  输入通道 1 → 中间通道 1                                   │
│  └───┘  ... 各通道独立处理，不混合                                 │
│                                                                  │
│  参数量: C_in × 1 × 3 × 3  (group=C_in)                          │
│  例如 64→64: 64×1×9 = 576                                        │
│                                                                  │
│  第 2 步: Pointwise (逐点)                                       │
│  ┌─┐                                                              │
│  │▓│  1×1 卷积核                                                  │
│  └─┘  将 C_in 个通道线性组合成 C_out 个通道                        │
│                                                                  │
│  参数量: C_in × C_out × 1 × 1                                    │
│  例如 64→128: 64×128×1 = 8,192                                   │
│                                                                  │
│  DSC 总参数量: 576 + 8,192 = 8,768                               │
│  标准卷积参数量: 73,728                                           │
│  压缩比: 8,768 / 73,728 ≈ 1/8.4 ≈ 节省 88%                      │
└──────────────────────────────────────────────────────────────────┘
```

**DSC 的完整 forward 流程：**

```
输入 x: [B, C_in, H, W]
  │
  ▼
Depthwise Conv2d(C_in→C_in, k3, groups=C_in)
  │  每个通道独立做 3×3 卷积
  │  只混合空间信息，不混合通道信息
  │  输出: [B, C_in, H, W]
  ▼
BatchNorm2d(C_in)        ← 标准化
  ▼
SiLU()                   ← 激活
  │  输出: [B, C_in, H, W]
  ▼
Pointwise Conv2d(C_in→C_out, k1)
  │  1×1 卷积混合所有通道
  │  每个输出通道 = 所有 C_in 个通道的加权线性和
  │  输出: [B, C_out, H, W]
  ▼
BatchNorm2d(C_out)       ← 标准化
  ▼
SiLU()                   ← 激活
  │
  ▼
输出: [B, C_out, H, W]
```

**为什么两步能替代一步？**

直觉理解：空间混合（Depthwise）+ 通道混合（Pointwise）分开做，效果 ≈ 同时做（标准卷积）。因为卷积本质就是"空间滤波 + 通道加权求和"，分开做意味着先滤波再看哪个通道重要，逻辑上是等价的。

### 2.5 SEBlock：通道注意力

```
SEBlock(channels=256, reduction=16)

输入: [B, 256, H, W]
  │
  ▼  Squeeze (压缩)
AdaptiveAvgPool2d(1)
  │  把每个通道的 H×W 空间信息"挤压"成一个数（全局平均值）
  │  [B, 256, H, W] → [B, 256, 1, 1] → view → [B, 256]
  │
  ▼  Excitation (激励)
Linear(256 → 256/16=16)    ─── 降维：256→16，压缩信息
  │  作用: 学习通道间的紧凑表示
  ▼
ReLU()                      ─── 引入非线性
  │
  ▼
Linear(16 → 256)            ─── 升维：16→256，恢复维度
  │  作用: 从紧凑表示中解码出每个通道的重要性
  ▼
Sigmoid()                   ─── 输出 0~1 之间的权重
  │  [B, 256] → view → [B, 256, 1, 1]
  │
  ▼  Scale (加权)
x = x × 权重
  │  每个通道乘以对应的权重
  │  重要通道 × ~1.0  → 保留
  │  噪声通道 × ~0.0  → 抑制
  │
  ▼
输出: [B, 256, H, W]
```

**SEBlock 的直观理解：**

```
假设输入是一张遥感图像经过网络后的 256 个特征图：

  通道 0: 检测边缘          → SE 说: 权重 0.92 (很重要，保留)
  通道 1: 检测圆形储罐      → SE 说: 权重 0.95 (场景可能有储罐)
  通道 3: 响应噪声/伪影     → SE 说: 权重 0.03 (噪声，忽略)
  通道 5: 检测道路纹理      → SE 说: 权重 0.88 (道路信息有用)
  ...
  通道 127: 死神经元        → SE 说: 权重 0.01 (完全没用，关掉)

SE 的核心作用: "看完全图后，告诉网络哪些特征通道值得关注"
```

**为什么 bottleneck 用 reduction=16？**

256→16→256 的瓶颈结构使 FC 参数量极少：
- 256×16 + 16×256 = 8,192 个参数
- 如果不用瓶颈（256→256→256）：256×256 + 256×256 = 131,072 个参数

用极少的参数（8K）换来信道级别的自适应特征选择，性价比极高。

### 2.6 GAP + 分类头：从特征图到分类结果

```
输入: [B, 512, 7, 7]   ← Stage 5 输出
  │
  ▼  GAP (Global Average Pooling)
AdaptiveAvgPool2d(1)
  │
  │  对每个通道的 7×7=49 个像素取平均值
  │  通道 0: avg(49个值) = 0.73
  │  通道 1: avg(49个值) = 0.21
  │  ...
  │  通道 511: avg(49个值) = 0.56
  │
  │  输出: [B, 512, 1, 1]
  │
  ▼  Flatten
  [B, 512]
  │  这就是图像的"特征向量"——用 512 个数描述一张遥感图像
  │
  ▼  Dropout(0.4)
  训练时随机丢弃 40% 的神经元
  │  防止过拟合: 模型不能依赖某几个特定神经元
  │
  ▼  Linear(512 → 256) + SiLU
  将 512 维特征压缩为 256 维
  │  参数: 512×256 + 256 = 131,328
  │
  ▼  Dropout(0.3)
  训练时随机丢弃 30% 的神经元
  │
  ▼  Linear(256 → 45)
  将 256 维特征映射为 45 个类别的分数
  │  参数: 256×45 + 45 = 11,565  (DOTA 15 类时: 256×15+15=3,855)
  │
  ▼
输出: [B, 45]
  │  softmax 后即为每个类别的预测概率
  │  取 argmax 得到预测类别
```

**为什么用 GAP 而不用 Flatten？**

```
Flatten 做法 (v1，导致 121M 参数):
  [B, 512, 7, 7] → Flatten → [B, 512×49=25088]
  → Linear(25088 → 4096)  ← 102M 参数! (占总参数 85%)
  → Linear(4096 → 4096)   ← 16.8M 参数
  → Linear(4096 → 15)     ← 61K 参数

GAP 做法 (v3，仅 2.76M 参数):
  [B, 512, 7, 7] → GAP → [B, 512]
  → Linear(512 → 256)     ← 131K 参数
  → Linear(256 → 15)      ← 3.8K 参数
```

GAP 的三个好处：
1. **参数量暴降 99%**：不需要巨大的全连接层
2. **防止过拟合**：没有大量参数需要训练，不会"记住"训练样本
3. **空间不变性**：无论目标在 7×7 网格的哪个位置，平均池化后结果相同

## 三、完整前向传播演练（以 DOTA 15 类为例）

```
输入: 一张被裁切出的 ship 图像 [1, 3, 224, 224]

┌─ Stem ─────────────────────────────────────────────────────┐
│ Conv2d(3→64, k3) + BN + SiLU                               │
│ → [1, 64, 224, 224]                                        │
│                                                             │
│ "将 RGB 颜色信息转化为 64 个基础纹理特征"                    │
│ 特征 0: 边缘分布   特征 1: 明暗对比                          │
│ 特征 2: 局部梯度   特征 3: RGB 线性组合                      │

## 五、关键设计决策问答

**Q: 为什么用 SiLU 而不是 ReLU？**

ReLU 对负值直接输出 0，导数也为 0，导致"神经元死亡"——一旦某个神经元对所有输入都输出 0，它永远无法恢复。SiLU（y = x·σ(x)）在负值区域保留微小的非零输出和梯度，给"濒死"神经元留一条恢复的路。

**Q: 为什么 Stage 3/4/5 用 2 个 ResidualBlock，Stage 1/2 只用 1 个？**

浅层特征图大（224×224, 112×112），计算量最大的地方应该少放层。深层特征图小（28×28, 14×14, 7×7），多加几层不会显著增加计算量，但能大幅提升特征抽象能力。

**Q: SEBlock 的 reduction=16 是怎么选的？**

经验值。太大（如 32）压缩过猛丢失信息，太小（如 4）起不到"去噪选优"的作用。16 是一个广泛验证的平衡点。

**Q: 为什么 Stage 5 通道不再翻倍（512→512 而不是 512→1024）？**

GAP 输出 512 维向量已经足够表达丰富的语义。继续翻倍会大幅增加 Stage 4 最后一层到 Stage 5 的参数量，但收益递减——512 通道已经能捕获足够多样的特征。
