# LSEVGG：轻量级遥感图像分类模型

## 摘要

遥感图像场景分类是计算机视觉领域的核心任务之一，其目标是将高分辨率航拍/卫星图像自动归类到预定义的场景类别中。本文提出了一种轻量级遥感图像分类模型 **LSEVGG**（Lightweight Squeeze-and-Excitation VGG），该模型将深度可分离卷积、SE 通道注意力机制、残差跳跃连接与 SiLU 激活函数有机结合，在保持极高分类精度的同时将参数量压缩至 2.76M。模型在 DOTA-v1.0（15 类目标切片）和 NWPU-RESISC45（45 类场景）两个遥感数据集上进行了训练与评估。实验结果表明：在 DOTA 数据集上，LSEVGG v3 以 2.76M 参数取得了 97.86% 的验证准确率，仅比 121M 参数的 v1 版本低 0.77 个百分点；在 NWPU-RESISC45 数据集上取得了 93.27% 的验证准确率。此外，本文还针对 DOTA 数据集中 small-vehicle 类别的高错误率进行了深入分析，揭示了切片归一化导致尺度信息丢失是其主要原因，并提出了相应的改进方向。

**关键词**：遥感图像分类；轻量级卷积神经网络；深度可分离卷积；SE 注意力机制；残差连接

## 一、引言

随着高分辨率遥感卫星和航空摄影技术的快速发展，海量遥感图像数据的自动解译成为遥感领域的重要需求。遥感图像场景分类旨在将航空或卫星图像自动分配到预定义的语义类别中，在城市规划、环境监测、军事侦察等领域具有广泛应用。

然而，遥感图像分类面临三个核心挑战：（1）**类间相似性高**——不同场景类别可能共享相似的纹理和结构，如港口与工业区在高空视角下难以区分；（2）**类内差异性大**——同一类别因拍摄角度、季节和光照条件的差异呈现出显著不同的外观；（3）**细粒度特征需求**——区分相似类别需要捕捉细微的局部纹理差异，这对模型的表示学习能力提出了较高要求。

近年来，深度学习在遥感图像分类领域取得了显著进展。VGG、ResNet、EfficientNet 等经典架构被广泛应用于遥感场景分类任务。然而，这些模型往往面临着精度与效率的权衡：高精度模型参数量庞大（通常 >100M），难以部署在资源受限的边缘设备上；轻量级模型虽然参数量小，但在复杂遥感场景下精度下降明显。

针对上述问题，本文设计并实现了 **LSEVGG**（Lightweight SE-VGG）模型，核心思想是通过深度可分离卷积降低计算量，通过 SE 通道注意力增强特征判别力，通过残差跳跃连接保障梯度流动，通过 GAP（全局平均池化）分类头大幅压缩全连接层参数。最终在 DOTA-v1.0 和 NWPU-RESISC45 两个数据集上验证了模型的有效性。

## 二、数据描述

### 2.1 DOTA-v1.0 数据集

DOTA（Dataset for Object deTection in Aerial images）是一个面向旋转目标检测的大规模遥感数据集[1]。本文使用其目标检测标注信息，将每个目标实例从原始图像中裁切出来，构造为分类数据集。

**预处理流程：**

1. **目标切片**：解析 DOTA 的旋转框标注（四点坐标 + 类别名），计算轴对齐外接矩形，向外扩 10 像素以保留上下文信息
2. **尺寸统一**：将各尺寸的切片统一 resize 至 224×224 像素
3. **类别组织**：按 15 个类别名分文件夹存放，兼容 PyTorch `ImageFolder` 数据加载接口
4. **数据划分**：对每个类别分别按 70%/20%/10% 比例随机划分为训练集、验证集和测试集（随机种子 42）
5. **存储优化**：优先使用硬链接（`os.link`）以节省磁盘空间，失败时回退为文件复制

**最终统计**：训练集 24,317 张，验证集 6,947 张，测试集 3,474 张，总计 34,738 张 224×224 像素的 RGB 切片图像。15 个类别包括 plane、ship、storage-tank、baseball-diamond、tennis-court、basketball-court、ground-track-field、harbor、bridge、large-vehicle、small-vehicle、helicopter、roundabout、soccer-ball-field 和 swimming-pool。

### 2.2 NWPU-RESISC45 数据集

NWPU-RESISC45 是目前遥感场景分类领域使用最广泛的基准数据集之一[2]，由西北工业大学创建。该数据集包含 45 个场景类别，每类 700 张 256×256 像素的 RGB 图像，涵盖飞机场、海滩、桥梁、丛林、沙漠、森林、高速公路、港口、工业区、岛屿、湖泊、山脉、住宅区、河流、体育场等多样性场景。

**预处理流程：**

1. **数据划分**：按 60%/20%/20% 比例随机划分为训练集、验证集和测试集（随机种子 42）
2. **归一化参数统计**：在训练集 18,900 张图像上逐通道计算 RGB 均值 `[0.3678, 0.3808, 0.3435]` 和标准差 `[0.1453, 0.1356, 0.1321]`

**最终统计**：训练集 18,900 张（420 张/类），验证集 6,300 张（140 张/类），测试集 6,300 张（140 张/类）。该数据集类别均衡，但类间相似度高，对模型细粒度判别能力提出了更高要求。

为提升模型的泛化能力和旋转/光照不变性，训练阶段采用如下数据增强策略：

## 三、方法模型

### 3.1 整体架构

LSEVGG v3 采用类 VGG 的分阶段卷积架构，共 5 个特征提取阶段（Stage）后接一个全局平均池化（GAP）分类头。网络输入为 224×224×3 的 RGB 图像，输出为 num_classes 维分类 logits。整体结构如下：

```
输入 [B, 3, 224, 224]
  │
  ├── Stem: Conv2d(3→64, k3) + BN + SiLU
  │
  ├── Stage 1: ResidualBlock(64→64) + MaxPool(2) → [B, 64, 112, 112]
  ├── Stage 2: ResidualBlock(64→128) + MaxPool(2) → [B, 128, 56, 56]
  ├── Stage 3: ResidualBlock(128→256) ×2 + MaxPool(2) → [B, 256, 28, 28]
  ├── Stage 4: ResidualBlock(256→512) ×2 + MaxPool(2) → [B, 512, 14, 14]
  ├── Stage 5: ResidualBlock(512→512) ×2 + MaxPool(2) → [B, 512, 7, 7]
  │
  ├── GAP: AdaptiveAvgPool2d(1) → [B, 512]
  └── Classifier: Dropout(0.4) → Linear(512→256) → SiLU
                   → Dropout(0.3) → Linear(256→num_classes)
```

### 3.2 核心模块设计

**SEBlock（Squeeze-and-Excitation 通道注意力）**

通道注意力机制由 Hu 等人提出[3]，其核心思想是显式建模特征通道间的依赖关系。SEBlock 先将空间维度压缩为单个描述符（全局平均池化），再通过瓶颈结构（FC→ReLU→FC→Sigmoid）学习通道权重，最后将权重与原始特征逐通道相乘。该机制以极小的计算开销（约增加 2-3% 参数量）显著提升了特征通道的判别力。

**DepthwiseSeparableConv（深度可分离卷积）**

深度可分离卷积将标准卷积拆分为两个步骤：Depthwise 卷积（逐通道 3×3 卷积，groups=in_channels）和 Pointwise 卷积（跨通道 1×1 卷积）[4]。这种方式将计算量从 `K²·C_in·C_out·H·W` 降至 `K²·C_in·H·W + C_in·C_out·H·W`，约为标准卷积的 1/9（当 K=3 时）。每步卷积后接 BatchNorm 和 SiLU 激活函数。SiLU（Swish）函数保留了 ReLU 的大部分优点，同时以平滑的非单调形状避免了深层神经元的"死亡"问题[5]。

**ResidualBlock（残差块）**

每个残差块由两个深度可分离卷积、一个 SE 注意力模块和一个跳跃连接组成。跳跃连接通过恒等映射或 1×1 卷积（当通道数变化时）将输入直接加到输出上，为梯度提供了"高速通道"[6]。这使得即使网络较深，浅层参数也能收到有效的梯度信号，从而学习到更丰富的细粒度特征表示。

### 3.3 版本演进

| 版本 | 主要特征 | 参数量 | DOTA 准确率 |
|------|---------|--------|------------|
| v1 | 标准 VGG + 三层全连接分类头 | 121.38M | 98.63% |
| v3 | 残差连接 + SE + DSC + GAP 轻量分类头 | **2.76M** | **97.86%** |

v3 版本的参数量仅为 v1 的 2.3%，但保持了 v1 准确率的 99.2%（97.86% vs 98.63%）。参数量的最大削减来自：深度可分离卷积替换标准卷积（约 80% 削减）和 GAP 替换全连接分类头（约 99% 削减）。

### 3.4 训练策略

模型训练采用以下策略以确保稳定收敛和良好泛化：

- **AMP 混合精度训练**：利用 `torch.cuda.amp` 在 forward 阶段使用 fp16 计算，利用 RTX 4060 的 Tensor Core 加速，同时在精度敏感操作保持 fp32
- **Warmup 学习率预热**：前 5 个 epoch 学习率从 0 线性升温至初始值 0.01，避免训练初期梯度不稳定
- **余弦退火调度**：warmup 结束后，学习率按余弦曲线从 0.01 衰减至 1e-6，使训练后期以极小步长精细搜索最优解[7]
- **Label Smoothing**：交叉熵损失中标签平滑系数 ε=0.1，防止模型过度自信，降低过拟合风险[8]
- **Early Stopping**：验证集准确率连续 15 个 epoch 无提升时自动终止训练

**v1 与 v3 对比分析**：v3 以 1/44 的参数量实现了 v1 准确率的 99.2%，充分验证了轻量化设计的有效性。v3 训练时间略长于 v1（206.6 vs 197.5 min），这是因为深度可分离卷积虽然理论计算量更低，但在 GPU 上的实际加速受内存带宽和算子实现限制。

### 4.4 DOTA vs NWPU 对比分析

| 维度 | DOTA（15 类） | NWPU-RESISC45（45 类） |
|------|-------------|---------------------|
| 类别数量 | 15 | 45（3 倍） |
| 训练样本数 | 24,317 | 18,900 |
| 单类样本数 | 不均衡（数百至数千） | 均衡（每类 420） |
| 图像性质 | 目标切片（224×224） | 场景原图（256×256 resize） |
| 最佳验证准确率 | **97.86%** | **93.27%** |
| 训练总耗时 | 206.6 min | 177.4 min |
| 收敛速度 | Epoch 10 达 92% | Epoch 30 达 88% |

NWPU 验证准确率（93.27%）显著低于 DOTA（97.86%），原因包括：

1. **分类空间复杂度倍增**：从 15 选 1 提升至 45 选 1，随机猜测期望准确率从 6.7% 降至 2.2%。512 维特征空间需容纳 3 倍数量的决策超平面。

2. **类间语义相似性更高**：NWPU 中存在大量语义相近的类别群组——`dense_residential`/`medium_residential`/`sparse_residential` 仅建筑物密度不同；`freeway`/`railway`/`intersection` 均为线状交通设施；`circular_farmland`/`rectangular_farmland`/`terrace` 仅形状不同。在 224×224 分辨率下，这些类别的纹理高度相似。

3. **单类训练样本更少**：420 张/类（vs DOTA 数百至数千不等），对需要学习细粒度判别边界的类别对而言不够充分。

4. **任务性质不同**：DOTA 为"目标级"分类（已裁剪出单个目标，背景干扰少），NWPU 为"场景级"分类（需从整幅图像中理解全局结构，信噪比更低）。

### 4.5 DOTA 中 small-vehicle 高错误率专项分析

在 DOTA 数据集的 15 个类别中，`small-vehicle`（小型车辆）是分类错误率最高的类别。以下从四个层面进行原因分析：

#### 4.5.1 切片归一化导致尺度信息丢失

DOTA 预处理中，所有目标切片被统一 resize 至 224×224。原始 large-vehicle 在图像中约占 50×80 像素，而 small-vehicle 仅约 15×25 像素。经过 resize 后二者在输入尺度上完全一致，模型丧失了利用绝对尺寸区分二者的能力——而绝对尺寸恰恰是区分大/小型车辆最直观的特征。

#### 4.5.2 纹理特征高度重叠

小型车辆与大型车辆在切片中共有大量视觉特征：矩形轮廓、金属表面反射、道路背景（灰色沥青）。二者的差异主要体现在细节纹理（车窗排列、车顶结构、阴影模式），需要高层的特征抽象能力。

#### 4.5.3 小型目标信噪比极低

small-vehicle 原始像素仅 ~15×25，外扩 margin=10 后约 35×45 像素。从 35×45 放大至 224×224 需要约 5-6 倍的上采样，双线性/双三次插值不可避免地模糊目标边缘并引入伪影。此外，切片中有效目标信息占比低，背景噪声占比远高于 large-vehicle。

#### 4.5.4 主要混淆路径

| 被误判为 | 混淆原因 |
|---------|---------|
| `large-vehicle` | 尺度归一化后尺寸相同，纹理、形状高度重叠 |
| `ship` | 小型船只俯视外观与车辆几乎一致（矩形、金属色） |
| `plane` | 停机坪上的小型飞机俯视形状与车辆近似 |
| `storage-tank` | 局部视角下圆形储罐边缘与车辆矩形轮廓接近 |

#### 4.5.5 改进方向

1. **尺度嵌入**：在特征通道中嵌入原始目标的像素面积或外接矩形尺寸
2. **多尺度输入分支**：同时输入原始分辨率和大尺度上下文两个视图
3. **对比学习**：针对 small/large-vehicle 难分样本对构建对比损失
4. **增大切片外扩**：当前 margin=10，增大至 30-50 可包含相邻车道的大车作为尺度参照物

## 五、结论与展望

本文提出了 **LSEVGG**，一个面向遥感图像分类的轻量级卷积神经网络。模型通过深度可分离卷积减少计算冗余，通过 SE 通道注意力增强特征判别力，通过残差跳跃连接保障深层梯度流动，通过 GAP+Dropout 分类头大幅压缩全连接层参数。核心结论如下：

1. **精度-效率的极致平衡**：LSEVGG v3 仅用 2.76M 参数（v1 的 2.3%）在 DOTA 15 类分类任务上达到 97.86% 的验证准确率，在 NWPU-RESISC45 45 类任务上达到 93.27%。该参数规模完全适合部署在无人机载、星载等资源受限的边缘计算平台。

2. **轻量化设计的有效性验证**：深度可分离卷积 + GAP 分类头是最有效的参数压缩策略（>98% 削减），而 SE 注意力和残差连接是维持精度不降的关键手段。

3. **尺度信息的不可替代性**：对 small-vehicle 的深入分析揭示了统一 resize 预处理的隐患——抹除尺度先验会导致特定类别间区分度显著下降，为后续研究提供了有价值的切入点。

**未来工作方向**：

- **知识蒸馏**：以 v1（121M）作为教师模型，通过软标签蒸馏进一步提升 v3 的细粒度分类精度
- **多尺度架构**：设计多尺度特征融合机制，让模型同时感知局部目标细节和全局尺度上下文
- **自监督预训练**：利用海量无标注遥感图像进行对比学习预训练，提升有限标注样本条件下的泛化能力
- **模型部署**：将 LSEVGG v3 转换为 ONNX/TensorRT 格式，在 Jetson Nano/Orin 等边缘设备上验证推理性能
