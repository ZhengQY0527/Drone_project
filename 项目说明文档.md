# LSEVGG_drone 项目说明文档

本文档用于快速说明当前代码的配置参数、如何修改参数、模型输入输出格式、训练流程，以及评估结果是如何生成的。

## 1. 项目做什么

这个项目把 DOTA 原始遥感目标检测数据，先按目标裁剪成分类样本，再训练一个轻量级的 SE-VGG 风格分类模型，最后在测试集上输出分类报告、混淆矩阵和各类别准确率图。

整个流程分为三步：

1. 数据预处理：把 DOTA 标注框裁剪成单目标图像切片。
2. 模型训练：用切片数据训练 LSEVGG 分类器。
3. 结果评估：在测试集上计算分类指标并保存可视化结果。

## 2. 代码里有哪些参数

所有核心参数都集中在 [config.py](config.py) 中，修改这里最直接。

### 2.1 数据路径参数

这些参数控制原始数据和切片数据的路径：

- `raw_train_image_dir`：原始训练集图像目录
- `raw_train_label_dir`：原始训练集标注目录
- `raw_val_image_dir`：原始验证集图像目录
- `raw_val_label_dir`：原始验证集标注目录
- `slice_root`：切片后数据根目录
- `train_dir`：切片后训练集目录
- `val_dir`：切片后验证集目录
- `test_dir`：切片后测试集目录

### 2.2 类别参数

`class_names` 定义了 15 个类别的顺序，`num_classes` 自动等于类别数。

这个顺序非常重要，因为：

- `ImageFolder` 会按子文件夹名生成类别索引
- 模型最后一层输出维度由 `num_classes` 决定
- 评估时的 `classification_report` 和混淆矩阵都依赖这个顺序

如果你改了类别顺序，训练和评估的标签映射也会跟着变。

### 2.3 切片参数

这些参数控制在 [preprocess_dota.py](preprocess_dota.py) 里如何裁剪目标：

- `slice_size`：切片统一缩放到的输入尺寸，当前是 224
- `margin`：裁剪时在目标框外额外扩展的像素
- `min_area`：过滤过小目标区域的最小面积阈值

### 2.4 训练参数

这些参数控制在 [train.py](train.py) 里的训练行为：

- `batch_size`：每个 batch 的图片数量
- `epochs`：训练轮数
- `lr`：初始学习率
- `momentum`：SGD 动量
- `weight_decay`：权重衰减
- `label_smoothing`：标签平滑系数
- `warmup_epochs`：学习率预热轮数
- `lr_scheduler`：学习率调度策略标识，目前代码里主要使用余弦退火
- `early_stop_patience`：早停耐心轮数
- `device`：自动选择 `cuda` 或 `cpu`

### 2.5 输出目录参数

- `save_dir`：模型保存目录，最佳模型会保存到这里
- `log_dir`：日志目录，目前预留但未在训练脚本里进一步写日志文件

## 3. 参数怎么改

### 3.1 修改训练规模

如果显存不足，优先修改：

- `batch_size`：减小 batch size 是最直接的办法
- `slice_size`：如果允许降低输入分辨率，也可以减小输入尺寸

如果想训练更充分，可以调整：

- `epochs`：增大训练轮数
- `early_stop_patience`：避免过早停止

### 3.2 修改学习率相关参数

如果训练不稳定或者收敛太慢，可以调整：

- `lr`：初始学习率
- `warmup_epochs`：前期预热时长
- `weight_decay`：正则化强度
- `label_smoothing`：抑制过拟合和过度自信预测

### 3.3 修改输入归一化参数

归一化参数写在 [dataset.py](dataset.py) 中：

- `mean`
- `std`

如果你重新预处理了数据，建议重新统计均值和标准差，然后同步修改这里。

### 3.4 修改数据增强

[dataset.py](dataset.py) 里的 `train_transform` 决定训练时的数据增强策略。

可调项包括：

- 随机裁剪范围
- 水平/垂直翻转
- 旋转角度
- 颜色扰动强度

验证集和测试集建议保持确定性变换，不要加随机增强。

### 3.5 修改模型类别数

如果你要换数据集或类别集合，需要同时改：

- [config.py](config.py) 中的 `class_names`
- `num_classes` 会自动随之变化
- [model.py](model.py) 中 `build_model(num_classes=...)`

如果新类别数和旧权重不一致，可以用 `pretrained_path` 加载匹配的层，分类头会自动跳过不匹配参数。

## 4. 模型输入输出是什么

模型定义在 [model.py](model.py) 中，主干是 `LSEVGG`。

### 4.1 模型输入

输入是一个标准的图像张量，形状为：

$$
[B, 3, 224, 224]
$$

其中：

- `B` 是 batch size
- `3` 是 RGB 三通道
- `224 x 224` 是统一输入尺寸

在数据加载时，这个输入来自 [dataset.py](dataset.py) 的 `train_transform` 或 `eval_transform`。

### 4.2 模型输出

模型输出是每个类别的 logits，形状为：

$$
[B, 15]
$$

如果你修改了类别数，输出维度会变成：

$$
[B, num\_classes]
$$

这些输出还不是概率，训练时直接送入交叉熵损失函数即可；推理时通常对它做 `argmax`，得到预测类别索引。

### 4.3 模型内部结构概览

`LSEVGG` 主要由下面几部分组成：

- 5 个卷积特征提取 block
- 深度可分离卷积模块
- SE 通道注意力模块
- 自适应平均池化
- 三层全连接分类器

这意味着模型先提取局部视觉特征，再压缩成固定尺寸向量，最后完成分类。

## 5. 怎么训练

训练入口在 [train.py](train.py)。

### 5.1 训练前要先准备什么

训练前需要先把数据整理成 `ImageFolder` 可识别的目录结构：

```text
data_slices/
  train/
    plane/
    ship/
    ...
  val/
    plane/
    ship/
    ...
  test/
    plane/
    ship/
    ...
```

如果还没有这个结构，先运行 [preprocess_dota.py](preprocess_dota.py) 做切片和划分。

### 5.2 训练命令

直接运行：

```bash
python train.py
```

训练脚本会自动完成：

1. 读取训练集和验证集
2. 构建模型
3. 定义损失函数和优化器
4. 执行训练循环
5. 保存验证集最优模型到 `./checkpoints/best_model.pth`

### 5.3 训练时发生了什么

在每个 epoch 中，代码会执行以下步骤：

1. 如果还在 warmup 阶段，就逐步提高学习率
2. 进入训练模式 `model.train()`
3. 前向传播得到输出 logits
4. 用交叉熵计算损失
5. 反向传播并更新参数
6. 统计训练准确率
7. 切换到验证模式 `model.eval()`
8. 在验证集上计算验证准确率
9. 如果验证准确率提升，就保存最佳模型
10. 如果连续多轮没有提升，就触发早停

### 5.4 训练结果怎么判断

训练阶段最直接看两个值：

- `Train Loss`：越小通常越好
- `Val Acc`：验证集准确率，越高越好

最终真正使用的模型通常是验证集表现最好的那个，而不是最后一个 epoch 的模型。

## 6. 结果如何得出

评估入口在 [evaluate.py](evaluate.py)。

### 6.1 评估流程

运行：

```bash
python evaluate.py
```

评估脚本会：

1. 读取测试集
2. 加载 `./checkpoints/best_model.pth`
3. 对测试集逐批推理
4. 收集所有预测值和真实标签
5. 输出分类报告
6. 生成混淆矩阵图
7. 生成各类别准确率条形图

### 6.2 分类报告怎么来的

`classification_report` 来自 `sklearn.metrics`，它会根据所有预测结果计算：

- precision
- recall
- f1-score
- support

这些指标都是由“真实标签”和“预测标签”对比统计得到的。

### 6.3 混淆矩阵怎么来的

`confusion_matrix` 也是基于所有预测结果计算出来的：

- 行代表真实类别
- 列代表预测类别
- 对角线越大，说明分类越准
- 非对角线越多，说明类别之间容易混淆

程序会把混淆矩阵保存为：

- `confusion_matrix_dota.png`

### 6.4 各类别准确率怎么来的

代码会按类别分别筛选测试集中的样本，然后计算该类别预测正确的比例。

保存结果为：

- `class_accuracy_dota.png`

这张图适合用来观察模型对哪些类别更强、哪些类别更弱。

## 7. 数据预处理怎么做

预处理入口在 [preprocess_dota.py](preprocess_dota.py)。

### 7.1 输入是什么

输入是 DOTA 原始图像和标注：

- 图像目录：`./DOTA/train/images` 和 `./DOTA/val/images`
- 标注目录：`./DOTA/train/labelTxt` 和 `./DOTA/val/labelTxt`

### 7.2 预处理做了什么

代码会：

1. 读取原图
2. 解析四点旋转框标注
3. 计算外接矩形
4. 按 `margin` 外扩裁剪范围
5. 过滤掉太小的目标
6. 裁剪并缩放到 `slice_size`
7. 按类别保存到临时池 `./all_slices`
8. 再按类别划分为 train/val/test

### 7.3 输出是什么

输出是可以直接被 `ImageFolder` 读取的分类数据集目录。

## 8. 代码文件作用速览

- [config.py](config.py)：集中管理路径、类别、训练和切片参数
- [preprocess_dota.py](preprocess_dota.py)：把 DOTA 标注图裁剪成分类数据
- [dataset.py](dataset.py)：定义数据增强、归一化和 DataLoader
- [model.py](model.py)：定义 LSEVGG 模型和加载预训练权重的方法
- [train.py](train.py)：训练模型并保存最佳权重
- [evaluate.py](evaluate.py)：加载最佳权重并输出测试结果
- [utils.py](utils.py)：提供均值和标准差统计工具

## 9. 常见修改建议

1. 如果你改了数据集，先检查 `class_names`、路径和 `num_classes`。
2. 如果显存不够，先减 `batch_size`，再考虑减 `slice_size`。
3. 如果训练不稳定，调低 `lr`，或增加 `warmup_epochs`。
4. 如果想看数据是否合理，先运行预处理，再检查 `data_slices` 目录里的各类别数量。
5. 如果评估结果很差，先确认训练集、验证集、测试集的标签顺序是否一致。

## 10. 推荐运行顺序

```bash
python preprocess_dota.py
python train.py
python evaluate.py
```

如果你要重新统计归一化参数，可以额外运行 `utils.py` 里的 `compute_mean_std`，然后把结果同步到 [dataset.py](dataset.py)。
