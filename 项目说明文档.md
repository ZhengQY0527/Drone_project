# LSEVGG_drone 项目说明文档

本文档基于当前代码状态（2026-05-18），说明 LSEVGG 遥感图像分类项目的配置、训练、评估与可视化流程。

## 1. 项目概述

本项目将 DOTA-v1.0 遥感目标检测数据按目标框裁剪为分类样本，训练一个带 SE 注意力机制的轻量级 VGG 风格分类器（LSEVGG），并在测试集上生成论文级评估图表。

**完整流程：**
1. **数据预处理** — 解析 DOTA 旋转框标注，裁剪目标并划分子集
2. **模型训练** — 使用 SGD + CosineAnnealing + AMP 混合精度训练
3. **结果评估** — 生成 10 张论文级可视化图表 + 分类报告

## 3. 模型架构

### 3.1 LSEVGG 结构

模型包含 5 个特征提取 Block + 3 层全连接分类头：

```
Input: [B, 3, 224, 224]
  Block1: Conv2d(3→64) → DepthwiseSepConv(64→64) → SE(64) → MaxPool → [B, 64, 112, 112]
  Block2: DepthwiseSepConv(64→128) → DepthwiseSepConv(128→128) → SE(128) → MaxPool → [B, 128, 56, 56]
  Block3: DepthwiseSepConv(128→256)×3 → SE(256) → MaxPool → [B, 256, 28, 28]
  Block4: DepthwiseSepConv(256→512)×3 → SE(512) → MaxPool → [B, 512, 14, 14]
  Block5: DepthwiseSepConv(512→512)×3 → SE(512) → MaxPool → [B, 512, 7, 7]
  AdaptiveAvgPool2d(7×7) → Flatten → [B, 25088]
  Linear(25088→4096) → ReLU → Dropout(0.5)
  Linear(4096→4096) → ReLU → Dropout(0.5)
  Linear(4096→num_classes) → [B, num_classes]
```

**关键组件：**

- **DepthwiseSeparableConv**：深度可分离卷积，先按通道做 3×3 空间卷积，再用 1×1 融合通道，降低参数量
- **SEBlock**：Squeeze-and-Excitation 通道注意力，通过全局池化学习通道权重，增强重要特征、抑制噪声
- **权重初始化**：Conv2d 用 Kaiming Normal，Linear 用 Normal(0, 0.01)，BatchNorm 用 γ=1, β=0

### 3.2 输入输出

- **输入**：`[B, 3, 224, 224]`（RGB，224×224 统一尺寸）
- **输出**：`[B, num_classes]`（logits，非概率；推理时取 argmax 得到类别索引）

### 3.3 LSEVGG v3 架构（当前版本）

v3 在 v1 基础上引入三项改进：残差跳跃连接、SiLU 激活函数、GAP 轻量分类头。

**完整结构：**

```
Input: [B, 3, 224, 224]
  Stem:    Conv2d(3→64) → BN → SiLU
  Stage1:  [ResBlock(64→64)] ×1  → MaxPool → [64,  112×112]
  Stage2:  [ResBlock(64→128)] ×1 → MaxPool → [128, 56×56]
  Stage3:  [ResBlock(128→256) + ResBlock(256→256)] → MaxPool → [256, 28×28]
  Stage4:  [ResBlock(256→512) + ResBlock(512→512)] → MaxPool → [512, 14×14]
  Stage5:  [ResBlock(512→512) + ResBlock(512→512)] → MaxPool → [512, 7×7]
  GAP:     AdaptiveAvgPool2d(1) → [512, 1×1]
  Head:    Flatten → Dropout(0.4) → Linear(512→256) → SiLU → Dropout(0.3) → Linear(256→15)
Output: [B, 15]
```

**ResBlock 内部结构（每个 Stage 由 1~2 个 ResBlock 组成）：**

```
         ┌─ DSC → DSC → SE ─┐
    x ───┤                   ├─→ + ─→ SiLU ─→ out
         └─ skip (1×1 Conv) ─┘
```

当输入输出通道数不同时（如 64→128），跳跃连接用 1×1 卷积对齐维度；同通道时用恒等映射。

### 3.4 v1 与 v3 版本对比

| 维度 | v1（原始版） | v3（当前版） |
|------|-------------|-------------|
| **连接方式** | 纯 VGG 顺序堆叠，无跳跃连接 | 残差跳跃连接，梯度直达浅层 |
| **激活函数** | ReLU（负值截断为 0，会死神经元） | SiLU (Swish)：x·σ(x)，负值保留小梯度 |
| **分类头** | AdaptivePool(7×7) → Linear(25088→4096→4096→15) | GAP → Linear(512→256→15) |
| **分类头参数量** | ~119M（占总参数 98%） | ~135K |
| **总参数量** | ~121M | **~2.8M** |

**v1 → v3 三项改动的动机：**

| 改动 | 解决的问题 | 原理 |
|------|-----------|------|
| **残差连接** | 深层梯度衰减，浅层学不到细粒度特征 | 跳跃连接为梯度提供"高速通道"，跨过 2 层 DSC 直达 block 入口，5 个 Stage 堆叠不再导致梯度消失 |
| **SiLU 激活** | ReLU 负值梯度为 0，深层神经元"死亡" | SiLU(x)=x·σ(x) 在 x<0 时保留微小负梯度，神经元可恢复 |
| **GAP 分类头** | 119M FC 参数严重过拟合（如 small-vehicle 仅 38%） | 全局平均池化强制每通道学习有判别力的整体特征，参数量缩 1000 倍 |

## 4. 配置参数说明

所有参数集中在 `config.py` 的 `Config` 类中，当前默认值针对 RTX 4060 Laptop GPU (8GB) 优化。

### 4.1 数据路径

| 参数 | 默认值 | 说明 |
|------|--------|------|
| `raw_train_image_dir` | `./DOTA/train/images` | 原始训练图像目录 |
| `raw_train_label_dir` | `./DOTA/train/labelTxt` | 原始训练标注目录 |
| `raw_val_image_dir` | `./DOTA/val/images` | 原始验证图像目录 |
| `raw_val_label_dir` | `./DOTA/val/labelTxt` | 原始验证标注目录 |
| `slice_root` | `./data_slices` | 切片数据集根目录 |
| `train_dir` | `./data_slices/train` | 切片训练集目录 |
| `val_dir` | `./data_slices/val` | 切片验证集目录 |
| `test_dir` | `./data_slices/test` | 切片测试集目录 |

### 4.2 切片参数

| 参数 | 默认值 | 说明 |
|------|--------|------|
| `slice_size` | 224 | 切片统一缩放尺寸 |
| `margin` | 10 | 裁剪时外扩像素（保留上下文） |
| `min_area` | 1024 | 最小切片面积（过滤噪声小框） |

### 4.3 训练超参数

| 参数 | 默认值 | 说明 |
|------|--------|------|
| `batch_size` | 16 | 批大小，8GB 显存推荐值，OOM 时降至 8 |
| `epochs` | 50 | 总训练轮数，early_stop 会自动提前终止 |
| `lr` | 0.01 | 初始学习率（SGD） |
| `momentum` | 0.9 | SGD 动量 |
| `weight_decay` | 5e-4 | L2 正则化系数 |
| `label_smoothing` | 0.1 | 标签平滑（防止过度自信） |
| `warmup_epochs` | 5 | 学习率线性 warmup 轮数（必须 < epochs） |
| `lr_scheduler` | cosine | 学习率调度策略：cosine 余弦退火 |
| `early_stop_patience` | 15 | 连续无提升轮数后触发早停 |
| `device` | auto | 自动选择 cuda/cpu |

### 4.4 输出目录

| 参数 | 默认值 | 说明 |
|------|--------|------|
| `save_dir` | `./checkpoints` | 模型检查点保存目录 |
| `log_dir` | `./logs` | 训练日志 + history.json 保存目录 |

### 4.5 类别定义

15 个 DOTA-v1.0 类别，顺序固定：plane, ship, storage-tank, baseball-diamond, tennis-court, basketball-court, ground-track-field, harbor, bridge, large-vehicle, small-vehicle, helicopter, roundabout, soccer-ball-field, swimming-pool。

## 5. 训练流程

### 5.1 前置条件

确保 `data_slices/` 下已有 `ImageFolder` 结构：

```
data_slices/
  train/plane/  train/ship/  ...
  val/plane/    val/ship/    ...
  test/plane/   test/ship/   ...
```

如果没有，先运行 `python preprocess_dota.py`。

### 5.2 启动训练

```bash
python train.py
```

### 5.3 训练过程详情

每个 epoch 自动执行：

1. **Warmup 阶段**（前 `warmup_epochs` 轮）：学习率从 0 线性升至 `lr`
2. **训练阶段**：AMP 混合精度前向 → loss 反向 → GradScaler 更新
3. **验证阶段**：无梯度推理，计算 val_loss 和 val_acc
4. **学习率调度**：warmup 结束后按 CosineAnnealing 从 `lr` 衰减到 `1e-6`
5. **模型保存**：val_acc 提升时保存完整 checkpoint（含 optimizer 和 scheduler 状态）
6. **早停检测**：连续 `early_stop_patience` 轮无提升则终止

### 5.4 训练时输出的文件

| 文件 | 位置 | 内容 |
|------|------|------|
| `train_YYYYMMDD_HHMMSS.log` | `./logs/` | 完整训练日志（时间戳、超参数、每轮指标、模型保存记录） |
| `history.json` | `./logs/` | 每轮 train_loss / train_acc / val_loss / val_acc / lr（JSON 格式） |
| `best_model.pth` | `./checkpoints/` | 验证集最佳模型的完整检查点 |

**查看实时日志：**
```bash
tail -f ./logs/train_*.log
```

## 7. 数据预处理

### 7.1 预处理流程

```bash
python preprocess_dota.py
```

脚本会：
1. 读取 DOTA 原始图像和四点旋转框标注
2. 跳过 `difficult=1` 的困难样本
3. 计算旋转框的外接矩形，按 `margin` 外扩
4. 过滤面积 < `min_area` 的过小目标
5. 裁剪并缩放到 `slice_size × slice_size`
6. 按类别保存到临时池 `./all_slices`
7. 按 70%/20%/10% 划分为 train/val/test
8. 统计并打印每类样本数量

### 7.2 输出结构

```
data_slices/
  train/  (70%)
  val/    (20%)
  test/   (10%)
```

每个子目录下按类别分子文件夹，可直接被 `torchvision.datasets.ImageFolder` 读取。

## 8. 数据集归一化参数

`dataset.py` 中的 `mean` 和 `std` 用于归一化。如需重新统计：

```python
from utils import compute_mean_std
mean, std = compute_mean_std("./data_slices")
```

然后将结果同步更新到 `dataset.py` 的 `mean` 和 `std` 变量。

训练集增强策略（`train_transform`）：随机裁剪（scale 0.8~1.0）、水平/垂直翻转、±15° 旋转、颜色抖动（亮度/对比度/饱和度 ±0.2，色相 ±0.1）。

## 9. 常见调整场景

### 显存不足 (CUDA OOM)
- 减小 `batch_size`（8 → 4）
- 减小 `slice_size`（224 → 192），但需重新预处理数据

### 模型过拟合
- 增大 `weight_decay`（5e-4 → 1e-3）
- 减小 `label_smoothing` 或保持不变
- 增强数据增强强度（增大 `RandomRotation`、`ColorJitter` 参数）

### 收敛太慢
- 增大 `lr`（0.01 → 0.05）
- 增大 `batch_size`（如果显存允许）
- 减少 `warmup_epochs`

### 更换数据集
- 修改 `config.py` 中的 `class_names`、路径参数
- 重新运行预处理
- 重新计算 `mean/std` 并更新 `dataset.py`

## 10. 推荐运行顺序
